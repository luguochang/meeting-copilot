## 二、综合决策与覆盖

skill 自动生成的系统有 2 处不适用，按 SKILL.md 允许的"页面覆盖"模式修正：

### 覆盖 1：字体（重要）
- **自动生成**：Cinzel + Josefin Sans（mood: real estate / luxury / elegant）— 完全不适用于技术产品
- **覆盖为**：`Lexend`（标题/UI）+ 系统中文栈（`PingFang SC` / `Microsoft YaHei` / `Noto Sans SC`）+ `Fira Code`（mono，用于时间戳/技术实体/token）
- **理由**：① Lexend 出自 skill 的 "Corporate Trust" 结果，明确"为可读性设计，a11y 优秀"；② 中文必须用系统栈，不加载中文 web font（动辄数 MB，严重拖慢首屏，违反 ux 域 Font Loading 规则）；③ Fira Code 出自 "Dashboard Data" 结果，适合数据/代码场景，用于技术实体高亮和时间戳

### 覆盖 2：背景色
- **自动生成**：纯黑 #000000（OLED）
- **覆盖为**：`--bg-deep: #0B0F1A` / `--bg-base: #0F172A` / `--bg-elevated: #11182C`（深蓝灰，非纯黑）
- **理由**：会议是长时场景（30-90 分钟），纯黑 + 高对比霓虹加速眼疲劳；深蓝灰保留"深色专业感"同时降低对比应激。这与 skill 的 `color-dark-mode` 规则一致（"desaturated / lighter tonal variants, not inverted"）

### 采纳：色彩与语义
直接采纳 skill 的琥珀主色 + 靛蓝强调，并扩展为语义状态色，**映射到会议状态机**（product-requirements 第 5 节）：

| 令牌 | 值 | 语义 |
|---|---|---|
| `--primary` | #D97706 琥珀 | 时间/注意力/进行中（录音脉冲） |
| `--accent` | #6366F1 靛蓝 | LLM/链接/CTA |
| `--ok` | #22C55E 绿 | 已确认决策 |
| `--warn` | #F59E0B 琥珀 | 候选/待确认 |
| `--info` | #38BDF8 青 | 行动项 |
| `--risk` | #EF4444 红 | 风险/破坏性/rollback 缺失 |
| `--approach` | #a78bfa 靛紫 | 方案考量卡（区别于缺口卡的状态色） |

5 种缺口卡片（owner/deadline/rollback/test/monitoring）用左边框色区分，满足 `color-not-decorative-only`（颜色 + 图标 + 文字三重编码）。

### 方案考量卡（v1 新增，区别于缺口卡）

原产品只做"缺口雷达"（漏了什么），v1 新增"方案考量卡"（正在讨论的方案怎么样）——填补"会议助手"能力空白。两类卡同结构、同管道，仅靠**左边框色 + 图标 + type 标签**区分：

| 维度 | 缺口卡 | 方案考量卡 |
|---|---|---|
| 左边框 | 状态色（红/琥珀/绿/青/靛） | 统一靛紫 #a78bfa |
| 图标 | 三角警告/人/时钟等 | 灯泡 |
| type 标签 | "缺口 · X" | "方案考量 · 替代档位/增量回归" |
| 对象 | 漏掉的项（owner/deadline/rollback…） | 方案内容（替代方案/考量点） |
| 语气 | "建议确认 owner" | "是否考虑过加 50% 档" |

mockup 演示 2 张：`approach.alternative`（灰度档位替代）、`approach.consideration`（增量回归考量），均带证据原文。详见优化计划 Phase 2.5。

## 三、信息架构（核心修复）

这是相对原 `index.html` 最重要的改变。原工作台主列堆了 15 个 section，其中 8 个是 no-op 流程面板。本方案：

```
顶栏  │ [●录音中] 议题 · 时间          [暂停] [结束并生成纪要]
──────┼──────────────────────────────────────────────────────
      │  状态板        实时转写流              当前建议/证据
      │  ──────        ──────────              ──────────
      │  当前议题       14:30  转写行           建议详情
      │  决策(2)        14:31  转写行           证据片段
      │  行动(3)        [建议卡片 inline]       LLM 追溯
      │  风险(1)        14:32  转写行
      │  待办(2)        [建议卡片 inline]
──────┴──────────────────────────────────────────────────────
状态条 │ ASR CER 8.2% · 召回 0.86✓   LLM 12次   延迟 P50 1.2s   [⚙系统状态]
```

**关键决策**：
1. **3 区工作台**（左状态板 / 中转写流 / 右详情）—— 转写流是主轴，建议卡片**内联在转写流的对应时间戳处**，而非独立列表。这呼应 product-requirements："主视图以会议状态和建议卡片为中心，transcript 作为可展开证据面板"。
2. **8 个 no-op 面板降级**为底部状态条的"⚙ 系统状态"按钮 → 打开右侧调试抽屉，默认隐藏。开发者可见，用户不可见。抽屉里明确标注 `noop_only`/`not_exited`/`disabled` 状态，并提示"Phase 2 打通后逐个变 ready"——把项目真相可视化但不打扰用户。
3. **工具栏从 8 个按钮收到 2 个**（暂停 / 结束并生成纪要）—— 原来的"工件主线/闭环预览/Synthetic Preview/真实模拟/长会模拟"等内部黑话全部移除。
4. **状态条替代 8 个面板**：ASR 质量 / LLM 调用 / 延迟 三组关键指标 + 调试入口，一行搞定。

## 四、交互与可访问性（遵循 skill Quick Reference）

| 规则 | 落地 |
|---|---|
| `excessive-motion`（每视图 ≤2 动画） | 仅录音脉冲 + 卡片 hover 微上移，无其他装饰动画 |
| `focus-states` | `:focus-visible` 2px 靛蓝描边 |
| `reduced-motion` | `@media (prefers-reduced-motion: reduce)` 全局关闭动画/过渡 |
| `color-not-only` | 缺口类型用 边框色 + 图标 + 文字标签 三重编码 |
| `touch-target-size` / `cursor-pointer` | 所有按钮 ≥34px，可点击元素 `cursor: pointer` |
| `loading-states` | partial 转写行用次级色 + `…` 表示进行中（骨架思路） |
| `primary-action` | 仅"结束并生成纪要"是主 CTA（靛蓝实心），其余次级 |
| `no-emoji-icons` | 全部 SVG 内联图标（Lucide 风格，1px-2px 描边统一） |
| `truncation-strategy` | 证据片段完整引用不截断 |
| `toast-dismiss` | Toast 2.2s 自动消失，`aria-live="polite"` |
| `whitespace-balance` / `spacing-scale` | 8dp 节奏（4/8/12/16/24/32/48） |

## 五、对 product-requirements 验收点的覆盖

| 验收点 | 方案体现 |
|---|---|
| 建议卡片每场 3-8 条 | mockup 演示 5 条（3 缺口 + 2 方案考量：rollback owner / 盯盘 owner / deadline 冲突 / 灰度档位 / 增量回归） |
| 每张卡片引用证据时间戳 + 原文 | 每张卡片底部 evidence 块含 `14:31 · "原文"` |
| 卡片在 final segment 后 10-30s 出现 | 内联在转写流对应时间戳后，视觉上即"相关 segment 后出现" |
| 工程语境门禁 | 状态板仅显示工程状态（决策/行动/风险/未闭环），非工程会议这些为 0 |
| 建议语气"可能缺少/建议确认" | 缺口卡用"建议确认""待确认"；方案考量卡用"是否考虑过"，均无"必须/不可上线" |
| 卡片可保留/忽略/复制/标记误报 | sug-actions 4 个 chip |
| 用户能看到云端调用状态 | 状态条 LLM 调用次数 + token，右栏 LLM 追溯（provider/model/prompt version/token） |
| 证据可跳回片段 | 右栏证据片段列表带 segment id |
