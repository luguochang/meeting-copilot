## 1. P0 — 真实端到端打通 + 错误修复（证明产品真能跑）

> 顺序：先把真实组件接通跑一遍，暴露并修掉每个真实错误，不允许用 fake 掩盖。

### P0-1 修复 browser_smoke E2E（被 /desktop 删除打断）
- 选项 A：把 browser_smoke 改为测 `/workbench` + 真实主链路（加载会议→events→LLM 卡→方案考量），不再依赖 /desktop/*。
- 选项 B：删除 browser_smoke，新写一个轻量 E2E 测 /workbench 主链路。
- 自测：`node code/web_mvp/e2e/browser_smoke.mjs` 退出 0。

### P0-2 LLM 容错 + 真实端到端验证
- llm_service 加 retry（指数退避，最多 2 次）+ 降级（网关 502/503/超时时返回 `degraded` 卡片或跳过，不抛 500）。
- 真实端到端：mock session → 真实 LLM → 真实卡片（网关恢复后跑一次，记录 token）。
- 自测：单元测试 retry/降级逻辑；网关恢复后真实调用产出 card_status=new。

### P0-3 真实麦克风采集端到端
- mic_capture.py sounddevice 真录音（5s）→ WS /live/asr/stream/ws/{id} → sherpa sidecar → 事件。
- 处理 macOS 麦克风权限（首次运行授权）。
- 自测：本机实跑，SSE/WS 收到真实转写 partial/final（非空文本）。

### P0-4 真实 LLM 端到端（网关恢复后）
- 真实 ASR 事件 → 真实 LLM 执行 → 真实卡片引用 active evidence + 真实用量。
- 自测：一次真实调用，card.llm_trace.usage.total_tokens > 0，card.suggestion_text 非空。

### P0-5 app.py 死代码清理
- 删除不再被任何路由调用的脚手架 helper + tool-loader（`_desktop_*`、`_load_*_module`、`_llm_card_lifecycle_*`、`_validate_llm_*` 等）。
