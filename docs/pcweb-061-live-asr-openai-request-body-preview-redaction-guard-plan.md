# PCWEB-061 Live ASR OpenAI Request Body Preview Redaction Guard Plan

## Goal

Add a local redaction guard to the PCWEB-060 OpenAI-compatible request body preview path.

PCWEB-060 intentionally copies selected `llm_request_draft_event.payload` fields into future chat-completions `messages` and `metadata`. Current drafts are generated locally and do not contain provider config or credentials. A future draft producer, however, could accidentally put provider/config/secret-like content into fields such as `suggested_prompt`, `input_summary`, `request_origin`, `source_event_ids`, `evidence_span_ids`, or `segment_batch`.

PCWEB-061 prevents that content from being reflected by the request body preview endpoint. It does not mutate the original Live ASR audit record, does not change `/llm-request-drafts`, does not read secrets, and does not call LLM.

## Endpoint

Existing endpoint:

```http
GET /live/asr/sessions/{session_id}/llm-openai-request-body-previews
```

## Redaction Rules

The preview path must sanitize string values before they are copied into:

- user message content.
- preview metadata string lists.
- preview metadata string values.

The guard detects only local, deterministic high-risk patterns:

- API-key-like token: `sk-...`.
- bearer token phrase: `Bearer ...`.
- authorization header marker: `Authorization:`.
- raw config marker: `raw_config`.
- API key marker: `api_key`.
- base URL marker: `base_url`.
- config path marker: `config_path`.
- local config directory marker: `configs/local`.
- provider relay-domain marker.

Detected string values are replaced with:

```text
[redacted:sensitive_draft_value]
```

The endpoint must also return redaction audit metadata:

```json
{
  "redaction_policy": "local_sensitive_draft_value_guard.v1",
  "redaction_status": "applied",
  "redacted_preview_count": 1
}
```

Each affected preview must include:

```json
{
  "redaction_status": "applied",
  "redacted_fields": [
    "suggested_prompt",
    "input_summary",
    "request_origin",
    "source_event_ids",
    "evidence_span_ids",
    "segment_batch"
  ]
}
```

If no values are redacted, top-level and per-preview status must be:

```json
{
  "redaction_status": "not_needed",
  "redacted_preview_count": 0,
  "redacted_fields": []
}
```

## Boundaries

- Read only.
- No mutation of original Live ASR audit events.
- No mutation of `/live/asr/sessions/{session_id}/llm-request-drafts`.
- No provider config read.
- No environment secret read.
- No keychain, credential manager, enterprise secret provider, or secret adapter access.
- No `configs/local/` read.
- No file existence/readability/size/mtime/hash/fingerprint check.
- No LLM gateway load.
- No network request.
- No token estimate or cost accounting.
- No schema generation or validation.
- No suggestion card or silenced event creation.
- No attempt to validate whether a secret is real. Pattern detection is only a local reflection guard.

## Why This Matters

PCWEB-060 made the future OpenAI-compatible request body visible. That is useful, but it also becomes the first place where draft text can be reflected into a provider-shaped prompt. PCWEB-061 adds a guardrail before any real provider execution exists, keeping the project aligned with the privacy rule that secrets and provider config must not enter prompts, reports, audit records generated by preview endpoints, or browser-visible responses.

This is intentionally smaller than a full DLP subsystem. It is a deterministic safety net for known high-risk markers until the real enabled executor, config reader, and logging redaction design are built.

## Tests

- A persisted Live ASR record with a hand-crafted `llm_request_draft_event` containing secret-like values in `suggested_prompt`, `input_summary`, `request_origin`, `source_event_ids`, `evidence_span_ids`, and `segment_batch` returns one request body preview without reflecting those values.
- The preview user message and metadata contain `[redacted:sensitive_draft_value]` placeholders.
- Top-level response reports `redaction_policy=local_sensitive_draft_value_guard.v1`, `redaction_status=applied`, and `redacted_preview_count=1`.
- The affected preview reports `redaction_status=applied` and stable `redacted_fields`.
- `/llm-request-drafts` still returns the original draft payload unchanged.
- `/events` still returns the original Live ASR events unchanged.
- A normal PCWEB-060 session keeps `redaction_status=not_needed`, `redacted_preview_count=0`, and empty per-preview `redacted_fields`.
- The guard does not read config files, environment secrets, keychain, or call LLM.
