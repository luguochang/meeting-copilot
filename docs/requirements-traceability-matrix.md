## 1. 不可违反边界

这些边界优先级高于具体实现方案：

- `BOUNDARY-001`：MVP 首发只承诺 macOS Apple Silicon；Windows 是第二阶段。
- `BOUNDARY-002`：默认 ASR 链路不得产生远程 API 费用；远程 ASR 默认关闭，只能显式配置启用。
- `BOUNDARY-003`：默认远程费用只允许来自 OpenAI-compatible LLM 中转站。
- `BOUNDARY-004`：产品主价值是实时/准实时会议状态和工程建议卡片，不是录音转文字或会后总结。
- `BOUNDARY-005`：所有正式状态、建议和纪要必须引用 EvidenceSpan；无证据内容只能作为草稿或待确认。
- `BOUNDARY-006`：非软件工程会议工程建议卡必须为 0。
- `BOUNDARY-007`：没有项目资料导入时，系统不得声称基于代码库或项目事实做核查。
- `BOUNDARY-008`：ASR 低质量、LLM 超时、schema 失败、证据链不完整时必须降级，不能输出强建议。
- `BOUNDARY-009`：真实录音、API key、模型缓存、运行输出不得进入仓库。

当前接口：

- `GET /`：打开本地 Web 工作台。
- `GET /static/app.js`、`GET /static/styles.css`：工作台静态资源。
- `GET /demo/fixtures`：列出可加载的 demo fixture。
- `POST /demo/fixtures/{fixture_id}/sessions`：从 fixture 创建 session，仍然走 `repo.create()` 和 core gate。
- `GET /sessions/{session_id}/events`：从 gated snapshot 生成 replay event timeline。
- `GET /sessions/{session_id}/events.sse`：同一事件流的 SSE 格式输出，供后续真实实时流替换。
- `POST /live/mock/fixtures/{fixture_id}/sessions`：从 fixture 创建 gated session，并生成 mock live event envelope。
- `GET /live/sessions/{session_id}/events`：读取 mock live JSON event stream。
- `GET /live/sessions/{session_id}/events.sse`：读取 mock live SSE event stream。
- `POST /live/asr/mock/sessions`：从 synthetic ASR streaming event JSON 创建本地 Live ASR audit session，不读取真实音频、不加载模型、不调用远程 provider。
- `GET /live/asr/sessions/{session_id}/events`：读取本地 Live ASR JSON event stream，包含 transcript、EvidenceSpan、本地四类 state lane skeleton、no-LLM scheduler audit、provider error 和 terminal summary。
- `GET /live/asr/sessions/{session_id}/events.sse`：读取同一 Live ASR event stream 的 SSE 格式输出，供 Web 工作台用 `EventSource` 增量消费。
- `GET /live/asr/sessions/{session_id}/suggestion-candidates`：只读查询 Live ASR no-LLM suggestion candidate queue，保留原始事件顺序和 payload。
- `GET /live/asr/sessions/{session_id}/llm-request-drafts`：只读查询 Live ASR no-LLM LLM request draft queue，保留原始事件顺序和 payload，但不执行请求。
- `GET /live/asr/sessions/{session_id}/llm-openai-request-body-previews`：`PCWEB-060` 只读查询从 no-LLM request draft 派生的 OpenAI-compatible chat-completions request body preview，固定 messages/response_format/metadata/idempotency 形状，但不读取 provider config、base URL、API key、authorization header、环境 secret、keychain 或 `configs/local/`，不调用中转站。

当前契约口径：

- `partial` 只用于实时预览，不生成正式 EvidenceSpan，不触发 LLM。
- `final` 进入统一 provider transcript `segments`，可生成 EvidenceSpan，但仍受 LLM 调度冷却和预算限制。
- `revision` 会替换被修正的 final segment，并按稳定更新进入调度。
- `error/end_of_stream` 进入 provider raw 运行状态，不生成正式 EvidenceSpan。
- 空 final 不触发 LLM。
- unknown event type 直接失败，避免 provider 适配错误被静默吞掉。
- mock streaming provider 是契约/调度回归工具，不代表 ASR 质量达标。
