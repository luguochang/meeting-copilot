# 文档归档

这里保存历史方案、阶段性计划和已经被后续主线文档取代的报告。

归档文档保留原始结论和日期，用于追溯项目为何做出当前决策；它们不是当前实现状态的唯一依据，也不能替代 `docs/current-mainline-index.md`、`docs/README.md` 或 `docs/handoff/` 中的交接说明。

## 目录

- `readiness-index.md`：历史 readiness 文档索引。
- `legacy/2026-07/`：2026 年 7 月早期方案、交付计划和阶段性报告。

## 读取规则

1. 先读当前主线索引和决策日志，再按需要查阅归档。
2. 归档中的路径、端口、测试数字和完成状态可能对应旧提交，不能直接作为当前发布结论。
3. 新需求、架构决策和验收结果应写入当前 `docs/` 体系，不要继续在归档目录追加新计划。
