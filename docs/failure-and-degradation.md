# 失败与降级策略

> 日期：2026-06-18  
> 目的：实时产品不能只设计成功路径。失败时要保住数据、信任和可复盘性。

## 1. 降级等级

```text
Level 0 正常：实时 ASR + 状态机 + 建议卡片 + 会后纪要
Level 1 轻微降级：实时 ASR 正常，建议卡片减少
Level 2 中度降级：停止实时建议，只保留转写和状态候选
Level 3 重度降级：停止 ASR，继续本地录音，稍后转写
Level 4 停止：音频采集失败，提示用户并保存诊断
```

## 2. 音频采集失败

症状：

- 麦克风无输入。
- 系统音频无输入。
- 设备切换。
- 丢帧。

处理：

- UI 明确提示。
- 记录失败时间段。
- 尝试重新枚举设备。
- 不生成该时间段的强建议。

## 3. ASR 延迟过高

触发：

- partial P95 > 2.5s。
- final P95 > 3.5s。
- 连续空转写。

处理：

- 降级到 Level 2。
- 建议卡片暂停。
- 保留音频用于会后重跑。
- 报告中标记低质量时间段。

## 4. ASR 质量低

触发：

- 关键技术实体置信度低。
- 连续术语识别错误。
- 用户频繁纠错。
- 音频质量差。

处理：

- 降低建议触发权重。
- 技术实体标记待确认。
- 强建议转为会后待确认项。

## 5. LLM 超时或 JSON 失败

处理：

- 不阻塞音频和 ASR。
- 将 transcript segment 入队。
- 稍后重试。
- JSON schema 失败不展示建议卡片。
- 会后报告标记失败批次。

## 6. Provider 断线

处理：

- 样本级或时间段级记录失败。
- 若配置备用 provider，可人工切换。
- MVP 不做复杂自动路由。
- 保留音频 chunk，恢复后补算。

## 7. 证据链不完整

触发：

- 缺时间戳。
- transcript revision 覆盖了原始片段。
- LLM 输出无 evidence span。

处理：

- 不进入正式纪要。
- 标记为待确认。
- 不生成强建议。

## 8. 用户纠错

用户可以纠正：

- 技术实体。
- owner。
- deadline。
- 决策状态。
- 建议误报。

处理：

- 用户纠错优先级高于模型。
- 纠错生成评测样本。
- 本场会议后续建议调整权重。
