# PCWEB-111 ASR Event Provenance Manifest Plan

> 日期：2026-07-03  
> 状态：Implemented  
> 主线节点：`EvidenceSpan/state/gap`、`candidate/card/feedback`、`pilot`  
> 边界：本计划不授权访问麦克风、不读取真实用户音频、不读取 `.m4a`、不读取 `data/asr_eval/local_samples/`、不读取 `configs/local/`、不调用远程 ASR/LLM、不下载模型、不下载公开音频、不运行 Cargo/Tauri。

## 1. 目标

PCWEB-110 已把 approved synthetic/mock ASR event file replay 成 timeline report，但 report 中的 `input_source_kind` 固定为 `approved_synthetic_event_file`。这会阻碍后续闭环审计：未来 replay 可能来自 `synthetic_audio`、`mock_streaming` 或经过人工证据校验的 `public_audio_sample`，需要在不读取真实音频、不下载公开音频、不调用 ASR/LLM 的前提下记录来源。

PCWEB-111 增加可选 ASR event provenance manifest。Replay 可以继续只读 events JSON；如果提供 manifest，工具必须先校验 manifest 的路径、schema、events path 绑定和 side-effect flags，再把来源写入 report。

## 3. Manifest Contract

Manifest 必须在 `artifacts/tmp/asr_events` 下，并且必须是 JSON object。

必填字段：

- `manifest_version=asr_event_provenance.v1`
- `events_path`
- `input_source_kind`
- `provider_candidate`
- `event_contract_version`
- `generated_by`
- `safe_to_call_llm=false`
- `safe_to_call_remote_asr=false`
- `safe_to_capture_microphone=false`
- `safe_to_read_user_audio=false`
- `safe_to_download_public_audio=false`

允许的 `input_source_kind`：

- `approved_synthetic_event_file`
- `synthetic_audio`
- `mock_streaming`
- `public_audio_sample`

可选来源字段：

- `source_id`
- `script_id`
- `sample_id`

Manifest 中的 `events_path` 必须与 replay 的 `events_path` 完全一致。任何 side-effect flag 为 true、manifest path 指向 forbidden root、manifest path 不在 approved events root、manifest 非 object 或 schema 不匹配，都必须 blocked。

## 4. 验收

成功路径：

- `build_asr_live_pipeline_replay_report(..., event_manifest_path=...)` 支持读取 provenance manifest。
- report 输出 `event_manifest_status=loaded`。
- report 输出 repo-relative `event_manifest_path`。
- report 的 `input_source_kind` 来自 manifest。
- report 输出 sanitized `event_provenance`，包含 manifest version、events path、input source kind、source/script/sample id、provider candidate、event contract version 和 generated by。
- report 不泄漏本机绝对路径。
- 所有原有 no-LLM/no-remote/no-mic safety flags 保持 false。

阻断路径：

- Manifest path 位于 `configs/local` 等 forbidden root 时，返回 `blocked_by_event_manifest_path_validation`，且读取前阻断。
- Manifest 任一 side-effect flag 不为 false 时，返回 `blocked_by_event_manifest_validation`。
- Manifest 的 provenance id fields 不得包含本机路径文本，例如 `/Users/...`。
- Manifest 的 provenance id fields 不得包含相对 forbidden-root 文本或音频路径文本，例如 `configs/local/...`、`data/asr_eval/local_samples/...` 或 `.m4a`。
- Manifest blocked 时，不读取 events，不生成 EvidenceSpan/state/candidate。
- 未提供 manifest 时保持 PCWEB-110 默认行为：`input_source_kind=approved_synthetic_event_file`，`event_manifest_status=not_provided`。
