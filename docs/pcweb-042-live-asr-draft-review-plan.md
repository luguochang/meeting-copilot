# PCWEB-042 Live ASR Draft Review Plan

> Date: 2026-07-01  
> Scope: PC Web MVP, draft review generated from local Live ASR audit events only.

## Goal

Generate a local post-meeting draft review from a persisted `live_asr_stream` audit record. The draft must summarize transcript text, evidence spans, local state candidates, scheduler decisions, provider errors, and stream summary without calling LLM or pretending to be a formal gated meeting report.

## Why This Matters

`PCWEB-041` made Live ASR event records durable. The next product step is making those records useful after the meeting: a user should be able to inspect what the local live pipeline heard, what state candidates it extracted, and why the scheduler would or would not call the LLM.

This is deliberately a draft review, not a formal report:

- It does not use the core session snapshot gate.
- It does not create suggestion cards.
- It does not call an LLM.
- It does not claim ASR quality is production-ready.

## Requirements

- Add JSON endpoint:
  - `GET /live/asr/sessions/{session_id}/draft`
- Add Markdown endpoint:
  - `GET /live/asr/sessions/{session_id}/draft.md`
- Draft JSON must include:
  - `session_id`
  - `source=live_asr_stream`
  - `trace_kind=live_event`
  - `review_type=asr_live_draft`
  - `is_formal_report=false`
  - `llm_call_status=not_called`
  - `transcript_text`
  - `transcript_segments`
  - `evidence_spans`
  - `state_candidates`
  - `scheduler_decisions`
  - `provider_errors`
  - `evaluation_summary`
  - `warnings`
- Draft Markdown must clearly say it is a draft and include:
  - title with session id
  - boundary statement
  - transcript section
  - evidence section
  - state candidate section
  - scheduler decision section
  - stream summary section
- Unknown Live ASR sessions return 404.
- Unsafe ids return 422 in JSON persistence mode via the existing repository guard.
- No LLM/schema/suggestion-card fields are generated by this draft endpoint.

## Boundaries

Out of scope:

- formal gated session snapshot
- official Markdown report from `build_markdown_report`
- LLM analysis
- suggestion cards
- real ASR quality proof
- audio/chunk persistence
- UI wiring for draft display

## Design

Add `meeting_copilot_web_mvp/asr_live_report.py`:

- `build_asr_live_draft_review(record) -> dict[str, Any]`
- `render_asr_live_draft_markdown(review) -> str`

The builder consumes the existing event envelope:

- `transcript_partial` is excluded from final transcript text.
- `transcript_final` and `transcript_revision` become transcript segments.
- active EvidenceSpans come from final/revision payloads.
- superseded EvidenceSpans remain in evidence output with `status=superseded`.
- `state_event.payload.state_item` becomes a state candidate.
- `scheduler_event.payload` becomes a scheduler decision.
- `provider_error.payload` becomes provider errors.
- latest `evaluation_summary.payload` becomes stream summary.

`app.py` will expose the two endpoints by reading the existing ASR live repository.
