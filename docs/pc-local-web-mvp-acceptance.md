## 3. Demo Gate

PC-1 demo 必须能回答：

- 这场会议在讨论什么工程问题？
- 有哪些候选决策？
- 谁有行动项，截止时间是什么，证据在哪里？
- 哪些风险或未闭环问题需要当场追问？
- AI 的建议来自哪段原文？
- 如果 ASR/LLM 低质量，系统如何降级？

## 4. No-Go

出现以下任一情况，不进入 Mac Desktop Shell：

- 建议卡片没有证据链。
- 建议卡片不能追溯到状态变化和缺口规则。
- 建议卡片无法计算是否在 10-30 秒可追问窗口内出现。
- 非工程会议仍输出工程建议。
- 卡片太晚，只能当会后总结。
- UI 以 transcript 为中心，会议状态和建议卡片只是附属。
- LLM 调用无法追踪触发原因和成本。
- 低质量 ASR、LLM 超时、schema 失败或证据链不完整时仍输出强建议。
- PC-1 只用 replay/mock 结果证明了结构链路，却被误判为真实桌面 ASR/音频采集达标。
