# Meeting Copilot 功能地图

> 日期：2026-06-18  
> 目的：明确产品实际有哪些功能，哪些属于 MVP，哪些属于后续阶段，避免需求文档只有抽象方向。

## 1. 功能总览

```text
会议采集
  -> 音频分轨
  -> 实时转写
  -> 中文技术语义增强
  -> 实时缺口雷达
  -> 结构化会议状态
  -> 会后工程产物
  -> 导出/协作
  -> 隐私与治理
  -> 评测与质量闭环
```

## 2. MVP 功能

| 模块 | 功能 | 说明 |
|---|---|---|
| 桌面客户端 | 手动开始/暂停/停止会议 | 不做后台自动监听 |
| 权限引导 | 麦克风、系统音频权限提示 | macOS 首发；Windows later |
| 音频采集 | mic/system/mixed 分轨 | 每轨保留时间戳 |
| 状态监控 | 电平、静音、丢帧、设备变化 | 失败必须提示 |
| 实时 ASR | 中文实时 provider 接入 | 先通过 bake-off 选型；默认本地 ASR |
| ASR Router | 多 provider 可切换 | 防止单点模型锁死 |
| 转写稳定器 | partial/final/revision | 支持回滚修正 |
| 技术词库 | 服务名、接口、字段、缩写 | 用于 ASR 热词和后处理 |
| 中文归一化 | 标点、断句、中英混排、术语修正 | 默认必选；保留 raw transcript |
| 会议状态机 | 议题、决策、待办、风险、未闭环问题 | 实时更新 |
| 缺口雷达 | owner/deadline/rollback/test/compatibility/metric | 低频提醒 |
| 建议卡片 | 证据、建议追问、保留/dismiss | 每场 3-8 条 |
| 结构化纪要 | 背景、决策、待办、风险、未闭环问题 | 带证据时间戳 |
| 导出 | Markdown/JSON | 后续再接工具 |
| 本地会话管理 | 列表、详情、删除 | 删除级联 |
| LLM Gateway | OpenAI-compatible | 流式、JSON、错误归一化 |
| 隐私控制 | 手动开启、状态可见、删除 | 不做隐蔽录音 |
| ASR 评测 | 中文 bake-off 工具 | 实现前先过门槛 |

## 3. 第二阶段功能

| 模块 | 功能 |
|---|---|
| ASR 增强 | FunASR/SenseVoice/sherpa-onnx 的 streaming、热词和模型优化 |
| 会后高质量转写 | 大模型重跑、说话人分离、字级时间戳 |
| 工程协作 | GitHub/Linear/Jira 任务草稿 |
| 文档产物 | ADR/RFC 草稿、发布 checklist、测试计划 |
| MCP | 让 Cursor/Claude/IDE 查询会议决策和证据 |
| 项目上下文 | 手动导入 repo 文档、ADR、issue |
| 企业控制 | 数据保留、访问权限、审计日志 |

## 4. 后续高级功能

| 模块 | 功能 |
|---|---|
| 代码库事实核查 | 检查会议断言是否与代码/文档冲突 |
| 监控/告警上下文 | 接入 Sentry/Datadog/PagerDuty |
| 自动任务创建 | 人工确认后一键创建 issue |
| Bot/API 模式 | 类 Vexa 的会议 bot 入会 |
| 多会议洞察 | 周报、风险趋势、决策追踪 |
| 企业私有化 | 私有 ASR、私有 LLM、数据驻留 |

## 5. 明确不做

- 隐蔽录音。
- 默认全天监听。
- 默认自动加入所有日历会议。
- 未经确认自动创建 issue。
- 实时替用户做架构决策。
- 未经评测承诺中文实时识别质量。
- 把 Whisper 当中文默认主链路。
- 用 AI 输出替代正式法律/人事/合同结论。

## 6. 当前最关键的产品闸门

第一道闸门不是 UI，也不是 LLM，而是中文技术会议 ASR。

Go 条件：

- 中文 CER <= 12%。
- raw 技术实体准确率必须单独记录，不能被 normalized 结果掩盖。
- normalized 后技术实体准确率 >= 85%，核心实体目标 >= 90%。
- 热词后核心术语准确率 >= 90%。
- final 延迟 P95 <= 3.5s。

No-Go：

- 中文技术词识别不稳。
- ASR 错误严重污染建议。
- LLM 修正经常编造。
- 不能保留证据链。

如果 ASR bake-off 不通过，MVP 应收缩为会后结构化纪要，不做实时 Copilot。
